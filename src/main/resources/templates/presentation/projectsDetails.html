개발 배경

연 평균 여행객은 점점 늘어나고 있습니다. 그러나 여행 계획을 세우는 것에 평균 20시간 이상을 투자하며 숙소, 티켓, 관광지를 선택하는데 많은 스트레스를 받으며 거짓 후기 및 바이럴마케팅 증가로 후기와 다른 모습에 실망하는 경우가 많습니다.

여행 기간은 출발 날짜부터 최대 3주까지 선택 가능

datepicker 를 활용하여 여행 출발 날짜와 마지막 날짜를 선택 가능하게 하였으며 마지막 날짜는 출발 날짜부터 최대 3주까지 선택 가능하게 하였습니다

여행 출발 및 마지막 최소 날짜는 당일

여행 출발 최소 날짜는 오늘이며 마지막 날짜는 출발 날짜부터 선택 가능합니다

왼쪽 해시태그(동반) 1개 / 오른쪽 해시태그(콘셉트) 최대 4개 선택 가능

일정 메인 기록 페이지

  • 일정 저장 버튼으로 일정 저장

날짜 및 지역 수정 가능

Ajax 를 사용하여 날짜와 지역을 수정하면 하단 계획 입력창도 날짜 수에 맞춰서 변경되게 구현하였습니다

계획 입력 및 예약 여부 선택

JavaScript 를 사용하여 필수 입력 항목인 장소와 시간을 입력 후 체크 아이콘을 클릭하면 입력한 내용으로 변경되게 하였습니다

계획 수정

삭제도 물론 가능합니다! 마이페이지 일정 조회 및 수정 삭제 부분에서 보여드리겠습니다

작성한 일정 조회 - 마이페이지

작성한 일정 목록 조회

  • 여행 마지막 날짜가 조회한 날짜보다 지났으면 여행 상태는 완료 / 아니면 기대하는 중

작성한 일정 상세 조회

  • 일정 지역 수정, 삭제 / 계획(장소) 수정, 삭제
  • 예약 여부 변경

상세 수정 및 body 클릭 시 수정 내용 반영

계획을 클릭하면 배경이 하늘색으로 변하며 연필 아이콘을 입력하면 수정할 수 있는 입력창이 나타납니다 체크 아이콘을 클릭하거나 body 를 클릭해도 수정된 내용이 반영되게 하였으며 Ajax 를 사용하여 수정된 내용이 DB에 반영됩니다

삭제

Ajax 를 사용하여 휴지통 아이콘 클릭 시 계획이 삭제되게 구현하였습니다

장소 추가

상세 조회에서 장소 추가 시 체크 아이콘을 클릭하지 않으면 추가한 계획은 저장되지 않습니다

예약 여부 변경

Ajax 를 사용하여 작성한 일정이어도 예약 여부 변경이 가능하게 구현하였습니다

플래너_일정 요청 상세 보기, 요청 진행 및 거절

플래너_일정 요청 상세 보기

요청 리스트에서 진행 상태가 일정 미작성이라면 진행 또는 거절을 할 수 있게 하였습니다

플래너_일정 요청 - 진행(일정 작성)

사용자가 신청한 날짜만큼 입력할 수 있게 생성되며 장소 추가, 수정, 삭제, body 클릭 전부 가능하게 하였으며 추가로 신청자에게 보낼 메모를 일정 하단에 입력할 수 있게 하였습니다

플래너_일정 요청 - 거절

거절 버튼을 클릭하면 모달창이 나타나 사유를 입력하고 취소하기 버튼을 누르면 진행 상태는 신청 취소로 변경되게 하였습니다
